    • Задание 3

      3. Сортируемая табличка
      3 балла
      Используя полученные знания по ES2015 и приемы функционального программирования реализуйте тестовое задание курса с сортируемой таблицей.
      Описание задания:
      Реализуйте веб-страницу с поиском по фильмам. На странице должен находиться поле для поиска и таблица. После ввода поискового запроса в поле для поиска, должны получаться данные для отображения с использованием The Movie Database API. После того, как данные получены - они должны отобразиться в таблице с такими полями (минималный набор, можно расширить): ID, Title, Language, Popularity Index, Votes Count, Rating, Release Date.
      При клике на заголовок столбца, все данные в таблице должны сортироваться по значениям в этом столбце по возрастанию, при втором клике - по убыванию, при третьем - опять по возрастанию и т.д.
      При клике на заголовок другой колонки, данные должны пересортировываться по значениям в этой колонке аналогично. При этом, необходимо ввести идикатор порядка сортировки для выбранной колонки, например, стрелки вверх и вниз (↑ и ↓).
      Требования:
      Используйте метод fetch для совершения запросов к серверу
      НЕ используйте jQuery. Только vanilla JavaScript
      Запрещено использовать сторонние библиотеки для сортировки, этот функционал нужно реализовать самостоятельно
      Постарайтесь реализовать механизм "сортируемости" таблицы максимально переиспользуемым. Так, чтобы его работа не зависела от данных, которые отображаются.
      Готовую, рабочую веб-страницу нужно разместить на Github Pages и приложить ссылку
      Для стилей можно использовать Bootstrap.

    • Задание 4

      4. Tweet - дополнительные параметры

      4.1 isLiked и isRetweeted

      Добавить к написанному на занятии компоненту твита параметры isLiked, isRetweeted. Нужно подсвечивать соответствующие иконки, твит был liked или retweeted.

      4.2 Размер текста

      Если вы замечали, в твиттере в зависимости от количества лайков и ретвитов, изменяется размер текста в твите. Предлагаю сделать упрощенную версию такого поведения.

      Например, давайте рассчитаем условный показатель популярности твита:

      popularity = likes + retweets

      И теперь, в зависимости от его значения, будем изменять размер шрифта.

      Например, так:

      popularity ∈ (0; 100) => 16px

        popularity ∈ [100; 1000) => 18px

        popularity ∈ [1000; ∞) => 22px

      Пример использования: <p style={{ fontSize: 21 }}>{text}</p>.
